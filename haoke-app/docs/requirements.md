# 好客租房 App 需求文档

> 版本：v0.1  
> 生成日期：2026-06-09  
> 来源：基于当前 Flutter 项目代码结构、页面、路由、模型与服务接口分析整理  
> 适用范围：移动端租房 App，兼容 Android、iOS，并保留 Web/桌面 Flutter 构建能力

## 1. 项目概述

好客租房 App 是面向租客与房东的租房服务客户端，目标是提供房源浏览、筛选搜索、房源详情、预约看房、收藏、订单、电子合同、房源发布与房源管理等能力。

当前项目已完成 Flutter 应用基础框架、首页四 Tab、登录注册、用户资料、房源详情、房源发布、房源管理、个人中心功能入口、设置、本地化、网络服务与本地安全存储等基础能力。部分业务页面仍为静态数据或占位交互，需要在后续迭代中接入真实接口并补齐完整业务闭环。

## 2. 产品目标

1. 帮助租客快速发现、筛选并评估合适房源。
2. 帮助租客完成收藏、联系房东、预约看房、下单、签约等租房流程。
3. 帮助房东或房源发布者发布、管理和跟踪房源状态。
4. 建立用户账号、资料、身份认证、钱包、合同、订单等个人中心能力。
5. 支持中文和英文基础本地化，为后续国际化扩展留出结构。

## 3. 用户角色

### 3.1 游客

- 可浏览首页、找房列表、资讯、房源详情。
- 可进入登录/注册。
- 访问需要账号的能力时，应提示登录。

### 3.2 租客

- 可搜索、筛选、浏览房源。
- 可查看房源详情、收藏房源、联系房东、预约看房。
- 可查看看房记录、我的订单、我的收藏、电子合同、钱包等个人功能。
- 可维护个人资料、语言设置、隐私设置、安全设置。

### 3.3 房东/发布者

- 可发布房源。
- 可选择或新增小区。
- 可管理已发布房源，区分空置中和已出租状态。
- 后续应支持编辑、下架、重新上架、查看咨询和预约记录。

### 3.4 平台/运营

当前客户端未提供运营后台。后续如需要，可扩展房源审核、举报处理、订单仲裁、合同模板、用户认证审核等能力。

## 4. 当前技术与架构现状

### 4.1 技术栈

- Flutter / Dart。
- Provider：管理筛选、认证、语言等状态。
- Fluro：统一路由。
- Dio：HTTP 请求。
- flutter_secure_storage：Token、用户信息、登录状态等本地安全存储。
- image_picker：图片选择。
- flutter_localizations + 自定义 AppLocalizations：中英文文案。

### 4.2 应用入口

- `lib/main.dart` 挂载 `FilterModel`、`AuthProvider`、`LocaleProvider`。
- `lib/application.dart` 配置主题、路由、本地化、启动页。
- `lib/routes.dart` 和 `lib/routes_handlers.dart` 负责页面路由注册。

### 4.3 已接入的后端接口方向

当前代码中已出现以下接口：

- 注册：`POST /api/auth/register`
- 登录：`POST /api/auth/login`
- 退出：`POST /api/auth/logout`
- 刷新 Token：`POST /api/auth/refresh-token`
- 当前用户：`GET /api/user/me`
- 更新用户：`PUT /api/user/me`
- 上传头像：`POST /api/file/avatar`
- 解绑银行卡/手机号：`POST /api/user/unbindPhone`
- 小区分页：`POST /api/house/community/page`
- 城市树：`GET /api/house/city/tree`
- 新增小区：`POST /api/house/community/create`
- 发布房源：`POST /api/house/resource/create`
- 房源详情：`GET /api/house/resource/{roomId}`
- 房源分页：`POST /api/house/resource/page`

## 5. 功能范围

### 5.1 首页

当前状态：已实现页面结构。

需求：

- 展示顶部搜索栏，包含城市位置与地图入口。
- 展示轮播图。
- 展示快捷导航入口。
- 展示推荐房源。
- 展示资讯列表。
- 支持下拉刷新。

待补齐：

- 搜索入口当前使用 `Navigator.of(context).pushNamed('search')`，路由未在当前路由表中明确注册，需统一搜索页路由。
- 轮播、推荐房源、资讯目前偏静态或本地数据，后续应接入接口。
- 地图入口需要定义产品范围：地图找房、定位城市、附近房源。

### 5.2 找房页

当前状态：已实现搜索栏、筛选栏、筛选抽屉、房源列表。

需求：

- 用户可按区域、租金、出租方式、户型、排序等条件筛选房源。
- 房源列表应展示封面、标题、描述、标签、价格。
- 点击房源进入房源详情。
- 支持下拉刷新。

待补齐：

- 当前列表使用本地数据，筛选回调未实际查询接口。
- 搜索关键词、筛选条件与接口字段映射需补充。
- 分页加载、空状态、错误重试需完整接入。

### 5.3 房源详情

当前状态：已接入房源详情接口，并包含加载、失败、重试、收藏状态、联系房东、预约看房等 UI。

需求：

- 展示房源图片、标题、租金、标签、面积、楼层、户型、朝向、配套、描述。
- 支持展开/收起长描述。
- 支持分享链接复制。
- 支持收藏/取消收藏。
- 支持联系房东。
- 支持预约看房。
- 展示相似推荐或猜你喜欢。

待补齐：

- 收藏目前为页面本地状态，需要接入收藏接口。
- 联系房东与预约看房按钮暂无实际业务处理。
- 分享链接目前为固定地址，需要生成真实房源链接。
- 举报入口暂无处理流程。

### 5.4 登录注册与认证

当前状态：登录、注册页面与接口已接入，AuthProvider 维护登录状态。

需求：

- 用户可通过用户名和密码登录。
- 用户可注册账号，注册成功后自动进入首页。
- 表单需校验用户名、密码、确认密码。
- 登录后保存 Token、用户信息、登录状态。
- Token 失效时支持刷新 Token，刷新失败自动清理登录状态。

待补齐：

- “记住我”当前仅有 UI，未实现账号记忆。
- 忘记密码/验证码登录暂未实现。
- 注册字段较少，后续可补充手机号、邮箱、协议勾选。
- 中文文案文件存在编码显示异常，需统一修复编码与文案质量。

### 5.5 个人中心

当前状态：已实现个人中心页面、用户头部、设置入口、功能宫格和资讯区域。

需求：

- 展示用户头像、昵称、登录状态。
- 未登录时展示游客状态和登录引导。
- 已登录时可进入个人资料编辑。
- 提供看房记录、我的订单、我的收藏、身份认证、联系我们、电子合同、房屋管理、钱包入口。

待补齐：

- 各入口权限规则需明确：游客点击是否跳登录。
- 部分入口页面为静态数据或占位交互。

### 5.6 个人资料

当前状态：已实现资料查看与编辑方向，支持头像选择和上传。

需求：

- 用户可查看用户 ID、用户名、昵称、邮箱、手机号、头像、创建时间、更新时间等信息。
- 用户可编辑昵称、邮箱、手机号、头像等资料。
- 保存成功后更新本地用户缓存。

待补齐：

- 字段可编辑范围与后端约束需明确。
- 头像上传失败、权限拒绝、图片过大等异常处理需完善。

### 5.7 设置

当前状态：已有设置页及隐私设置、手机号绑定、语言设置、帮助中心、关于我们、修改密码等子页面。

需求：

- 支持账号与安全设置：修改密码、手机号绑定/解绑。
- 支持通知设置：推送通知、邮件通知。
- 支持隐私设置：资料可见性、手机号/邮箱展示、搜索与推荐、数据采集、广告个性化、清理记录、注销账号。
- 支持语言切换。
- 支持帮助中心和关于我们。
- 支持退出登录。

待补齐：

- 修改密码接口当前未实现，Provider 返回 `Change password API is not implemented`。
- 通知、隐私、多数安全设置当前偏本地 UI 状态，需接入后端。
- 注销账号、清理搜索记录、清理聊天记录需明确接口和安全确认流程。

### 5.8 看房记录

需求：

- 展示用户预约看房记录。
- 每条记录包含房源/小区名称、地址、预约时间、状态、联系人或备注。

待补齐：

- 支持取消预约、重新预约、联系经纪人/房东。

需求：

待补齐：

- 接入订单列表与订单详情接口。

需求：

## 7. 非功能需求

### 7.1 性能

- 首页、找房页和个人中心首屏应在合理网络条件下快速展示。
- 列表页需支持分页加载和下拉刷新。
- 图片需使用缓存加载，避免重复下载。

### 7.2 稳定性

- 所有网络请求需提供加载、成功、失败、重试或空状态。
- Token 过期需自动刷新，刷新失败需安全退出登录态。
- 表单提交需避免重复点击。

### 7.3 安全

- Token 和用户敏感信息使用安全存储。
- 身份认证、合同、钱包、手机号等敏感能力必须登录后访问。
- 证件信息、合同、支付相关数据需避免日志明文输出。

### 7.4 国际化

- 当前支持中文和英文。
- 新增页面必须补齐中英文 key。
- 需要修复当前中文文案编码异常，统一为 UTF-8。

### 7.5 可测试性

- 路由入口、功能按钮跳转、表单校验、Provider 状态变更应有 Widget 或单元测试。
- 关键业务流程需补充回归测试：登录、房源详情加载、发布房源、房源管理、预约看房。

## 9. 当前主要风险

1. 中文源文件或文案存在编码异常，会影响 UI 展示、测试断言和维护效率。
2. 多个业务页面仍为静态数据，用户流程未形成真实闭环。
3. 搜索路由疑似未注册，首页/找房页搜索入口可能无法正常跳转。
4. 发布房源表单字段与提交模型不一致，图片和描述等关键字段未提交。
5. 修改密码、隐私设置、通知设置、订单、合同、钱包等功能当前缺少真实接口。
6. 权限拦截策略尚不明确，游客可能进入需要登录的业务页面。
